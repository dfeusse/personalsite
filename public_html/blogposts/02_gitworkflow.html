Git and Github workflow for a two person team. Or just using two computers.

May 2013

I have seen infinite git and Gthub tutorials posted throughout the internet. I work on two computers, a personal Macbook and a machine running Windows for work. I wanted to post a basic, crystal clear list of commands people could follow for either working on two different computers or working with a partner. This assumes you have a Github account. Go to this account on github.com and create a new repository. My username is dfeusse, so saubsitute your useername for mine in the commands.
1. Make sure you have generated SSH keys. If this is unfamilar, follow the directions in this link. It is very simple.
2. On your first Windows machine, follow these commands after navigating into the correct directory using the git bash.
git init
got config core.autocrlf false
git remote add origin git@github.com:dfeusse/x.git
git pull origin master
git push origin master
3. Now go to your Macbook/second computer, or this can be the second person working on the same team. Follow these commands after navigating into the correct directory.
git init
git remote add origin git@github.com:dfeusse/x.git
git pull origin master
4. Now you can make any changes desired. After, you just have to push to master. This is how the files appear on Github. Then on the other machine, or the other person, can pull these changes, his directory will be updated, and he can do whatever work desired.
